🎯 一、網頁基本架構

請建立以下檔案：

index.html

style.css

script.js

使用 繁體中文介面、具備簡潔動態的 UI 與動畫效果。

🟦 二、功能需求（務必全部完成）  
1️⃣ 跑馬燈（Marquee）模組

可在介面上設定：

跑馬燈文字內容

字體顏色

背景顏色

速度

可插入表情符號與貼圖（以 Unicode 或 Emoji 顯示）

跑馬燈內容需支援即時更新（不需重新整理頁面）

2️⃣ 倒數計時器（Timer）

放在跑馬燈下方。功能：

可設定 倒數分鐘 與 秒數

按一下「開始倒數」即可啟動

具備：

時間到時響鈴或跳通知文字（可簡單）

可重新設定

3️⃣ YouTube 影片播放區

功能需求：

有一個欄位可貼入「YouTube 網址」

系統自動嵌入 iframe

支援播放與全螢幕

4️⃣ 隨機抽號碼機（含進階設定）

位置在影片下方。

抽號碼功能需要支援：

設定開始值（例如 1）

設定結束值（例如 50）

設定要排除的號碼（例如：5, 12, 30）

一次抽出多少個號碼（例如：3個）

抽出後顯示「動畫效果」

抽過的號碼可以：

全部清除

或避免重複抽取（可選）

5️⃣ 下方互動區（打賞／按讚）

需求：

能讓使用者按：

❤️ 讚

⭐ 星星

🚀 火箭（打賞效果）

每按一下要有動畫效果（例如炸裂或跳動）

要可以解決「其他使用者如何按」的問題 →  
✔ 請使用 Firebase Realtime Database / Firestore 來同步所有使用者的點擊數  
✔ 使用者從不同電腦／手機按讚，都能即時同步顯示

6️⃣ 左上角「活動標題動畫」

功能：

一個可輸入「課程／活動名稱」的文字欄位

顯示於左上角

需具備以下動畫擇一：

打字機效果

漸層流動

漂浮動態

7️⃣ 主持人資訊（社群連結）

需要支援：

Facebook 連結

YouTube 連結

IG（如果可補充）

以圖示（icon button）呈現，點擊後打開新分頁

🟧 三、技術要求

介面具備 RWD（手機、平板、筆電皆可使用）

所有控制項需排列整齊、美觀

動畫請利用 CSS 或 JS（不要太重）

可使用：

Firebase（處理多人按讚）

localStorage（儲存跑馬燈設定）
